// how it works

Every request in four steps

Most requests never reach OpenAI. They're served in milliseconds from Qdrant.

🔑 01 — Fingerprint

Namespace the cache

SHA-256 hash the system prompt, combine with tenant ID. Completely isolates each user's cache from others.

🔢 02 — Embed

Vectorize the query

The user turn is passed through text-embedding-3-small producing a 1536-dimensional semantic vector.

🔍 03 — Search

ANN lookup in Qdrant

Approximate nearest-neighbor search finds the closest cached vector. Above the per-query-type threshold — serve from cache.

💾 04 — Store

Cache and return

On a miss: forward to OpenAI, store response with TTL. The next semantically equivalent query gets a hit.

// features

Built for production

Not a toy cache — engineering that makes this real.

🎯

Learned Threshold Calibration

A logistic regression classifier trained on (query_A, query_B, should_cache) pairs. ~15% better recall than a fixed cosine threshold, no extra false positives.

🏢

Multi-Tenancy

Every tenant's cache namespaced by {tenant_id}:{system_fingerprint}. Pass X-Tenant-ID — zero cross-contamination.

📡

Streaming SSE

Buffers streamed tokens, writes to cache, re-streams on hits. Works with "stream": true — the client never knows the difference.

⏱️

TTL Invalidation

Configurable TTL per entry. Expired entries evicted lazily on lookup. Per-namespace invalidation for topic-drift handling.

🚀

Cold-Start Pre-Warming

Historical logs clustered via k-means on embeddings. Representative queries pre-generated and seeded into the cache at launch.

🔌

Drop-in Compatible

Fully OpenAI-compatible. Change base_url, nothing else. Works with any SDK or HTTP client targeting /v1/chat/completions.
