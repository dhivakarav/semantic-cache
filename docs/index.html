// how it works

Request lifecycle

Every incoming LLM call passes through a 3-step pipeline before touching the upstream API.

🔑 STEP 01

Fingerprint

The system prompt is SHA-256 hashed. Combined with the tenant ID, this forms the cache namespace — preventing cross-tenant contamination.

🔢 STEP 02

Embed

The user turn is passed through text-embedding-3-small to produce a 1536-dimensional semantic vector.

🔍 STEP 03

ANN Lookup

Qdrant searches for the nearest cached vector. If cosine similarity exceeds the query-type threshold, the cached response is returned immediately.

⚡ STEP 04

Store or Serve

On a miss, the request is forwarded to OpenAI. The response is stored in Qdrant with a TTL, then returned to the client.

// features

What makes this non-trivial

Beyond the basic cache — the engineering problems that make this production-worthy.

🎯

Learned Threshold Calibration

A logistic regression classifier trained on (query_A, query_B, should_cache) pairs replaces fixed cosine thresholds. Outperforms fixed thresholds by ~15% recall with no increase in false positives.

🏢

Multi-Tenancy

Each tenant's cache is namespaced by {tenant_id}:{system_fingerprint}. Pass X-Tenant-ID header — one tenant's responses never contaminate another's.

📡

Streaming SSE

LLMs stream tokens. This proxy buffers streamed content, writes it to cache, and re-streams from cache on hits — the client never knows the difference. Works with "stream": true.

⏱️

TTL Invalidation

Every cached entry carries a configurable TTL. Expired entries are evicted lazily on lookup. Supports topic-drift detection via per-namespace invalidation API.

🚀

Cold-Start Pre-Warming

Historical query logs are clustered via k-means on their embeddings. The most representative query from each centroid is pre-generated, solving the empty-cache problem on launch.

🔌

Drop-in Replacement

Fully OpenAI-compatible. Change one line: point base_url at the proxy. No SDK changes, no prompt modifications. Works with any client that calls /v1/chat/completions.

// usage

Drop-in one-liner

Point your OpenAI client at the proxy. Everything else stays the same.

Python openai SDK 
from openai import OpenAI

client = OpenAI(
    api_key="your-key",
    base_url="http://localhost:8000/v1",  # ← only change
)

# First call — cache miss, hits OpenAI (~120ms)
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "What is Python?"}],
)

# Second call — cache hit, served from Qdrant (~4ms)
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Tell me about Python"}],
    extra_headers={"X-Tenant-ID": "org-123"},  # multi-tenant
)

bash Quick start 
# Clone and install
git clone https://github.com/dhivakarav/semantic-cache
cd semantic-cache
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# Configure
cp .env.example .env  # add OPENAI_API_KEY

# Run (Docker Compose)
OPENAI_API_KEY=sk-... docker-compose -f docker/docker-compose.yml up

# Run demo (no key needed)
python3 demo.py
